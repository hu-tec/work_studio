# 신청서 모듈화 설계 — work_studio

## 핵심 구조

신청서는 **3가지 대상** × **분류표 5축** 교차로 구성된다.

### 대상 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **일반인** | 수강/이용 신청자 | 수업신청, 레벨테스트, 설명회, 견적의뢰, 1:1문의 |
| **전문가** | 전문 서비스 제공자 | 번역사 신청, 전문가 지원, 매칭 등록, CAT도구 경력 |
| **교육자** | 강의/교육 제공자 | 강사 지원, 커리큘럼 제안, 강의분류(대중소급수) |

---

## 모듈 설계 총괄표

> **읽는 법**: 각 행은 하나의 모듈. ● = 필수 포함, ○ = 선택 포함, — = 미포함
> 
> 모듈은 **계층 1 (공통)** → **계층 2 (유형별)** → **계층 3 (분야/홈페이지별)** 순서로 조립된다.

| 계층 | 모듈 ID | 모듈명 | 일반인 | 전문가 | 교육자 | 필드 요약 | 분류표 축 |
|------|---------|--------|--------|--------|--------|-----------|-----------|
| **1. 공통** | `M-COM-01` | 기본 인적사항 | ● | ● | ● | 성명, 연락처, 이메일 | — |
| **1. 공통** | `M-COM-02` | 기본 자격사항 | ● | ● | ● | 활동시간, 경력, 학력, 통화가능, 직업, 지원목적 | — |
| **1. 공통** | `M-COM-03` | 동의/비고 | ● | ● | ● | 개인정보동의, 자기소개, 비고(포트폴리오) | — |
| **2. 유형** | `M-TYP-USR` | 일반 이용정보 | ● | — | — | 서비스종류, 희망일정, 예산, 요청사항 | — |
| **2. 유형** | `M-TYP-EXP` | 전문가 프로필 | — | ● | — | 전문분야, 경력상세, 포트폴리오, 이력서(파일) | — |
| **2. 유형** | `M-TYP-EDU` | 교육자 프로필 | — | — | ● | 강의경력, 교수역량, 비대면도구, 인증/자격증 | — |
| **3. 분야** | `M-FLD-LANG` | 언어/번역 전문 | — | ● | ○ | 가능언어(11), 번역분야(13), CAT도구, 기계번역경험 | A분야(번역) |
| **3. 분야** | `M-FLD-TEACH` | 강의 분류체계 | — | — | ● | 대분류(TESOL/프롬/번역/윤리), 중분류(교육/일반/전문), 급수(1~8) | A분야 × B급수 |
| **3. 분야** | `M-FLD-SKILL-T` | 강사 스킬셋 | — | — | ● | AI교육도구, 프롬프트설계, 콘텐츠제작, LMS, 인터랙티브도구 | A분야 |
| **3. 분야** | `M-FLD-SKILL-E` | 전문가 스킬셋 | — | ● | — | AI Vibe Coding, 프롬프트활용, 디자인도구, 문서작성, 번역시스템 | A분야 |
| **3. 홈페이지** | `M-SITE-COURSE` | 과정/수업 선택 | ● | — | — | 과정(TESOL/프롬/AI번역/윤리), 신청유형, 지역, 일정 | C홈페이지 |
| **3. 홈페이지** | `M-SITE-QUOTE` | 견적 의뢰 | ● | — | — | 출발어, 도착어, 문서유형, 분량, 납기, 요청사항 | C홈페이지(번역) |
| **3. 홈페이지** | `M-SITE-EXHIBIT` | 전시/행사 신청 | ● | — | — | 전시유형, 장소, 일정, 인원 | C홈페이지(전시회) |
| **3. 홈페이지** | `M-SITE-MATCH` | 매칭 요청 | ● | — | — | 서비스유형, 예산, 기한, 요구사항 | C홈페이지(매칭) |
| **3. 홈페이지** | `M-SITE-EXAM` | 시험 신청 | ● | — | — | 시험종류, 급수, 지역, 일정 | C홈페이지(시험) |

## 조립 예시

### TESOL 수업신청 (일반인)
```
M-COM-01  기본 인적사항     ← 공통
M-SITE-COURSE  과정 선택   ← 홈페이지(TESOL)
M-COM-02  기본 자격사항     ← 공통
M-COM-03  동의/비고        ← 공통
```

### 강사 지원서 (교육자)
```
M-COM-01  기본 인적사항     ← 공통
M-FLD-TEACH  강의 분류체계  ← 분야×급수
M-COM-02  기본 자격사항     ← 공통
M-TYP-EDU  교육자 프로필    ← 유형
M-FLD-SKILL-T  강사 스킬셋  ← 분야
M-COM-03  동의/비고        ← 공통
```

### 번역사 신청 (전문가)
```
M-COM-01  기본 인적사항     ← 공통
M-COM-02  기본 자격사항     ← 공통
M-TYP-EXP  전문가 프로필    ← 유형
M-FLD-LANG  언어/번역 전문  ← 분야(번역)
M-FLD-SKILL-E  전문가 스킬  ← 분야
M-COM-03  동의/비고        ← 공통
```

### 견적 의뢰 (일반인, AI번역 사이트)
```
M-COM-01  기본 인적사항     ← 공통
M-TYP-USR  일반 이용정보    ← 유형
M-SITE-QUOTE  견적 상세    ← 홈페이지(번역)
M-COM-03  동의/비고        ← 공통
```

## 분류표 교차 참조

### 분야 × 분류 (A × A하위)

문제은행·커리큘럼·교재·신청서가 **모두 공유**하는 분류 체계:

| 대분류 | 중분류 | 소분류 예시 |
|--------|--------|-------------|
| 문서 | 비즈니스 | 사업계획서, 회사소개, PPT, 엑셀, 기획서 |
| 문서 | 법률 | 소송장, 준비서면, 형사, 민사 |
| 문서 | 의료/특허/노무/교재/논문/기사/고전 | — |
| 음성 | 아나운서/관광가이드/큐레이터/안내방송/교육강의/실시간/화상수업 | — |
| 영상/SNS | 미디어/장르 | 유튜브, 다큐멘터리, 영화, 드라마, 예능 |
| IT/개발 | 개발/보안 | AI, 에이전트, DB, 빅데이터, 백엔드, 프론트, 프로그램 |
| IT/개발 | 디자인/기획 | 웹모바일디자인, 웹기획, UIUX, 컨텐츠 |
| 창의적활동 | 콘텐츠 | 드라마, 웹툰소설, 소설, 시, 음악, 미술 |
| 번역 | 통번역 방식 | 순차통역, 동시통역, 음성번역, 자가선택 |
| 프롬프트 | — | — |
| 확장영역 | 라이프/전문 | 암, 요리, 재무, 주식, 부동산, ... |

### 분야 × 급수 (A × B)

| 분야 | 구분 | 급수 |
|------|------|------|
| 프롬프트 | 교육 | 1~8급 |
| 프롬프트 | 일반 | 1~3급 |
| 프롬프트 | 전문 | 1~2급 |
| 번역 | 교육 | 1~8급 |
| 번역 | 일반 | 1~3급 |
| 번역 | 전문 | 1~2급 |
| 윤리 | 교육 | 1~8급 |
| 윤리 | 일반 | 1~3급 |
| 윤리 | 전문 | 1~2급 |

## work_studio 콘텐츠 타입 통합 설계

> 신청서뿐 아니라 문제은행·커리큘럼·교재도 **같은 분류표 축**을 공유한다.
> work_studio에 4가지 콘텐츠 관리 기능을 둔다.

| 콘텐츠 타입 | DB 테이블 | 관리 탭 | 분류 축 | 핵심 필드 |
|-------------|-----------|---------|---------|-----------|
| **신청서** | `form_modules` + `form_templates` | 신청서 | C홈페이지 × 대상유형 | 모듈 조합, 필드 정의 |
| **문제은행** | `questions` | 문제은행 | A분야 × B급수 | 문제유형, 지문, 보기, 정답, 해설, 난이도, 배점 |
| **커리큘럼** | `curriculum` | 커리큘럼 | A분야 × B급수 | 주차, 주제, 학습목표, 학습활동, 평가방법, 교재연결 |
| **교재** | `textbooks` | 교재 | A분야 × B급수 | 단원, 제목, 내용요약, 파일(PDF/영상), 난이도, 페이지 |

### 공통 분류 인덱스 (모든 콘텐츠 공유)

모든 콘텐츠에 다음 메타데이터가 붙는다:

```
{
  _field_big: "문서",              // A분야 대분류
  _field_mid: "법률",              // A분야 중분류
  _field_sm: "소송장",             // A분야 소분류
  _level_area: "번역",             // B급수 분야
  _level_div: "교육",              // B급수 구분
  _level_grade: 3,                // B급수 급
  _site: "ai_trans",              // C홈페이지
  _dept: "마케팅",                 // D그룹 부서
  _team: "강사 팀",                // D그룹 팀
  _grade: "강사",                  // E등급
}
```

이 인덱스를 공유하면 **분야×급수 교차 필터**로 신청서·문제·커리큘럼·교재를 한번에 조회할 수 있다.

## 대시보드 개선 방향

현재 대시보드는 리스트(접수 데이터) 요약만 보여줌. 개선안:

| 영역 | 현재 | 개선 |
|------|------|------|
| **1행** | 접수 요약 (전체/오늘/신규/확인/완료) | 유지 + 신청서·문제·커리·교재 각 건수 추가 |
| **2행** | — | **4칸 요약 카드**: 활성 신청서 N개, 문제은행 N문제, 커리큘럼 N과정, 교재 N건 |
| **3행** | 홈페이지별 접수 카드 (4열) | 유지 (접수 현황) |
| **4행** | — | **분야×급수 히트맵**: 어느 분야/급수에 콘텐츠가 많은지 한눈에 |

## 구현 우선순위

| 순서 | 항목 | 설명 |
|------|------|------|
| 1 | 신청서 모듈 시스템 | ✅ 완료 (현재 상태) |
| 2 | 대시보드 개선 | 요약 카드 + 신청서/리스트 통합 뷰 |
| 3 | 문제은행 테이블 + 관리 UI | questions 테이블, 분야×급수 필터, CRUD |
| 4 | 커리큘럼 테이블 + 관리 UI | curriculum 테이블, 주차별 관리 |
| 5 | 교재 테이블 + 관리 UI | textbooks 테이블, 파일 업로드 |
| 6 | 분류표 통합 필터 | 5축 교차 필터로 모든 콘텐츠 검색 |
